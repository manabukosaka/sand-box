# ADR 0004: DuckDB 採用の最終決定 (Final Approval of DuckDB Selection)

## 1. ステータス (Status)
Accepted (Supersedes [ADR 0001](./0001-datastore-selection.md))

## 2. コンテキスト (Context)
[ADR 0001](./0001-datastore-selection.md) において、メインのデータストア候補として DuckDB が推奨されたが、最終決定は技術検証 (PoC) の結果を待つこととしていた。
2026年4月26日に実施された PoC (`docs/poc_report.md`) により、実効性が確認された。

## 3. 決定事項 (Decision)
データストアとして **DuckDB** を正式に採用する。

## 4. 決定の根拠 (Rationale)
1.  **書き込み性能の実証:** Rust + DuckDB (Appender API) とインメモリバッファリングを組み合わせることで、DB への直接 I/O 負荷を抑えつつ、高いスループットでのログ受信が可能であることが実証された。
2.  **運用性:** 単一バイナリへの組み込みが容易であり、外部 DB プロセスを必要としない軽量性がプロジェクトの目的に合致している。
3.  **分析クエリの適性:** 列指向アーキテクチャにより、将来的なメトリクス集計や大量ログ検索における高いパフォーマンスが期待できる。

## 5. 結果 (Consequences)
- 今後の MVP 開発において、バックエンドは DuckDB 接続を前提として実装を進める。
- インメモリバッファの堅牢性（プロセスダウン時のデータ保護など）については、継続的な設計課題とする。
