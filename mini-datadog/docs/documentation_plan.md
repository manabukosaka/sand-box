# ドキュメンテーション計画 (Documentation Plan)

## 概要
本ドキュメントは、Mini Datadog プロジェクトにおけるドキュメント整備の全体方針とロードマップを定義します。開発チーム、運用者、そしてエンドユーザーが迷わずシステムを理解・活用できるように、目的別にドキュメントを体系化します。

## 1. プロダクトマニュアル（ユーザー向け）
システムを導入し、日々の監視やログ分析に利用するエンドユーザー向けのドキュメント群です。

*   **クイックスタートガイド (Quick Start Guide)**
    *   単一バイナリ（またはDocker Compose）を用いた最短での起動手順。
    *   サンプルデータの送信とダッシュボードでの確認方法。
*   **セットアップ手順 (Setup Instructions)**
    *   本番環境へのデプロイメントガイド（Linux サーバーでの systemd 起動、環境変数の設定など）。
    *   データ保持期間（Retention）などの設定変更方法。
*   **主要機能の使い方 (User Guide)**
    *   **ログ検索機能:** タイムレンジ指定、キーワード検索、タグ（`service`, `env` 等）によるフィルタリングの活用方法。
    *   **Live Tail 機能:** リアルタイムログストリーミングの見方と活用シーン。
    *   **メトリクスダッシュボード:** CPU/メモリ使用量グラフの読み方とリソース監視のポイント。
*   **FAQ（よくある質問とトラブルシューティング）**
    *   「データが表示されない」「ディスク容量が逼迫している」「APIのレイテンシが高い」などの一般的な問題への対処法。

## 2. 技術ドキュメント（開発者・運用者向け）
システムの内部構造を理解し、拡張や運用保守を行うエンジニア向けのドキュメント群です。

*   **アーキテクチャ詳細 (Architecture Details)**
    *   システム全体の構成図（フロントエンドとバックエンドの通信フロー）。
    *   主要な技術選定の背景と決定事項（ADRへのリンク集）。
*   **APIリファレンス (API Reference)**
    *   ログ受信、メトリクス受信などの各種エンドポイントの仕様。
    *   OpenAPI (Swagger) 形式での定義ファイル提供と、各エンドポイントのリクエスト・レスポンス例。
*   **DBスキーマ詳細 (Database Schema)**
    *   DuckDBにおけるログデータおよびメトリクスデータのテーブル構造。
    *   時系列データに対するインデックス設計や、クリーンアップ処理の仕組み。
*   **環境構築ガイド (Development Environment Setup)**
    *   開発者向けのローカル環境構築手順（Rust / Next.js のセットアップ）。
    *   テスト実行手順と品質基準（Lint, フォーマッタ、カバレッジ）。
*   **拡張・コントリビューションガイド (Contribution Guide)**
    *   ブランチ戦略（GitHub Flow 等）、PR作成のルール、コミットメッセージ規約（Conventional Commits）。
    *   新規APIやUIコンポーネントを追加する際の設計ガイドライン。

## 3. ロードマップ (Documentation Roadmap)
ドキュメントの整備は、プロダクトの開発フェーズと連動して段階的に進めます。

### Phase 1〜2（完了済）
*   `requirements.md` (要件定義)
*   `architecture.md` (アーキテクチャ概要)
*   `api_schema.md` (APIドラフト)
*   `adr/*` (アーキテクチャ意思決定記録)
*   `poc_report.md` (PoC結果報告)

### Phase 3（MVP開発進行中）
*   基本設計・詳細設計のアップデート
*   ローカル開発用の `README.md` 整備（開発者向け環境構築のベース）

### Phase 4（アルファリリースとV&V）★重要整備フェーズ
アルファ版を利用してV&V（検証・妥当性確認）を行うため、**利用者がシステムを自律的に動かせる最低限のドキュメント**を完成させます。
*   **ドキュメント整備内容:**
    *   **クイックスタートガイド:** 単一バイナリによる配布と起動手順のドキュメント化 (`README.md` に統合、または独立ページ化)。
    *   **セットアップ手順の基礎:** 環境変数によるデータ保持期間の設定方法など、最低限の運用設定の明文化。
    *   **主要機能の使い方（基礎）:** V&Vテスターが機能（検索、Live Tail、メトリクス）を確認するためのマニュアル作成。
    *   **APIリファレンス（MVP版）:** データ送信用APIの仕様確定とドキュメント化（OpenAPI定義の作成開始）。
    *   **環境構築ガイドの拡充:** テストパイプラインやCI手順の追記。

### Phase 5（機能拡充と安定化・ベータ/正式リリース）
正式リリースに向け、包括的なドキュメントセットを完成させます。
*   **ドキュメント整備内容:**
    *   **本番運用向けセットアップ手順:** 高度な設定やセキュリティ要件（API認証等）の追記。
    *   **FAQ / トラブルシューティングの完成:** V&Vやドッグフーディングで得られた知見の反映。
    *   **拡張・コントリビューションガイドの完成:** オープンソース展開や社内引き継ぎを見据えたガイド整備。
    *   **DBスキーマ・アーキテクチャ詳細の最終化:** 実装との乖離をなくし、最新状態に更新。
